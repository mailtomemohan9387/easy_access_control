Easy Access Control

Control who can create, edit, or delete records in Odoo easily and safely.

User-Based Rules

Apply access control rules for individual users with a simple setup.

Model-Level Control

Restrict actions for specific models such as Contacts and other business records.

Safe Operations

Prevent accidental edits, unwanted record creation, and unauthorized deletion.

Problem it solves

In many Odoo systems, users may accidentally create, edit, or delete important business data. This can lead to data loss, confusion, and operational issues. Easy Access Control helps administrators prevent such problems by controlling user actions clearly and effectively.

Key Features

  • User-based access control
  • Model-level permission management
  • Block Create action
  • Block Edit action
  • Block Delete action
  • Simple rule setup and configuration
  • Lightweight and easy to use

How it works

  1. Create an access rule
  2. Select the user
  3. Select the target model
  4. Choose whether to allow or restrict Create, Edit, and Delete actions
  5. Save the rule and enforce permissions instantly

Use Cases

  • Prevent employees from deleting important contact records
  • Restrict junior users from editing master data
  • Allow safe usage without giving full modification rights
  • Improve data safety in daily Odoo operations

Compatibility

Works on Odoo.sh Works on On-Premise Not supported on Odoo Online

This module supports environments where custom modules can be installed, such as Odoo.sh and On-Premise deployments. It is not supported on Odoo Online because Odoo Online does not allow installation of custom third-party modules.
